## Середовища KSeF API 2.0 
15.12.2025

Нижче наведено інформацію про публічні середовища.

| Скорочення | Середовище                       | Опис                                                                 | Документація API                         | Допустимі формати                         |
|-------|----------------------------------|-----------------------------------|----------------------------------|----------------------------------------------|
| **TEST**  | Тестове <br/> (Release Candidate)        | Середовище для тестування інтеграції з KSeF API 2.0, містить версії RC. | https://api-test.ksef.mf.gov.pl/docs/v2   | FA(2), FA(3), FA_PEF (3), FA_KOR_PEF (3)     |
| **DEMO**  | Предпродуктивне (Demo)    | Середовище, що відповідає продуктивній конфігурації, призначене для фінальної валідації інтеграції в умовах, наближених до продуктивних. | https://api-demo.ksef.mf.gov.pl/docs/v2   | FA(3), FA_PEF (3), FA_KOR_PEF (3)            |
| **PRD** | Продуктивне                        | Середовище для виставлення та отримання рахунків-фактур з повною юридичною силою, з гарантованим SLA та відповідними продуктивними даними.                           | https://api.ksef.mf.gov.pl/docs/v2             | FA(3), FA_PEF (3), FA_KOR_PEF (3)            |



> <font color="red">Увага:</font> Тестові середовища (TE/DEMO) служать виключно для тестування інтеграції з API KSeF. Не слід надсилати до них **продуктивні рахунки-фактури** або реальні дані суб'єктів.

У тестовому середовищі `TE` допускається аутентифікація з використанням самопідписних сертифікатів (self-signed), що на практиці означає, що багато інтеграторів можуть [аутентифікуватися](uwierzytelnianie.md#proces-uwierzytelniania) в контексті тієї ж компанії.
З цієї причини дані, введені в середовищі `TE`, не є ізольованими і можуть бути спільними між інтеграторами.
Для тестів слід використовувати випадкові ідентифікатори NIP, уникаючи будь-яких реальних даних.

### Сервісні роботи на тестових середовищах
У зв'язку з плановим, систематичним розвитком Національної системи електронних рахунків-фактур (KSeF 2.0), **з 1 жовтня 2025 року** на тестових середовищах Системи можуть проводитися циклічні сервісні роботи.

Ці роботи будуть виконуватися в годинах з **16:00 до 18:00**. У цей час можуть виникнути тимчасові утруднення в доступі до тестових середовищ.

Після завершення робіт у [changelogu](api-changelog.md) будуть публікуватися **виключно зміни, що впливають на інтеграцію**, наприклад, зміни поведінки API, модифікації контрактів, схем XSD, лімітів тощо. Зміни, що не впливають на API і не помітні з перспективи інтеграції, наприклад, внутрішні виправлення якості, продуктивності, безпеки **можуть не повідомлятися** або будуть представлені зведено.
