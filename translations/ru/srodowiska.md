## Среды KSeF API 2.0 
15.12.2025

Ниже приведена информация о публичных средах.

| Аббр. | Среда                       | Описание                                                                 | Документация API                         | Допустимые форматы                         |
|-------|----------------------------------|-----------------------------------|----------------------------------|----------------------------------------------|
| **TEST**  | Тестовая <br/> (Release Candidate)        | Среда для тестирования интеграции с KSeF API 2.0, содержит версии RC. | https://api-test.ksef.mf.gov.pl/docs/v2   | FA(2), FA(3), FA_PEF (3), FA_KOR_PEF (3)     |
| **DEMO**  | Предпродуктовая (Demo)    | Среда, соответствующая продуктовой конфигурации, предназначенная для финальной валидации интеграции в условиях, приближенных к продуктовым. | https://api-demo.ksef.mf.gov.pl/docs/v2   | FA(3), FA_PEF (3), FA_KOR_PEF (3)            |
| **PRD** | Продуктовая                        | Среда для выставления и получения счетов-фактур с полной юридической силой, с гарантированным SLA и соответствующими продуктовыми данными.                           | https://api.ksef.mf.gov.pl/docs/v2             | FA(3), FA_PEF (3), FA_KOR_PEF (3)            |



> <font color="red">Внимание:</font> Тестовые среды (TE/DEMO) служат исключительно для тестирования интеграции с API KSeF. Не следует отправлять в них **продуктовые счета-фактуры** или реальные данные субъектов.

В тестовой среде `TE` допустима аутентификация с использованием самоподписанных сертификатов (self-signed), что на практике означает, что многие интеграторы могут [аутентифицироваться](uwierzytelnianie.md#proces-uwierzytelniania) в контексте одной и той же компании.
По этой причине данные, введенные в среде `TE`, не изолированы и могут быть общими между интеграторами.
Для тестов следует использовать случайные идентификаторы NIP, избегая любых реальных данных.

### Сервисные работы в тестовых средах
В связи с планируемым, систематическим развитием Национальной системы электронных счетов-фактур (KSeF 2.0), **с 1 октября 2025 г.** в тестовых средах Системы могут проводиться циклические сервисные работы.

Эти работы будут проводиться в часы с **16:00 до 18:00**. В это время могут возникать временные затруднения в доступе к тестовым средам.

После завершения работ в [changelog](api-changelog.md) будут публиковаться **исключительно изменения, влияющие на интеграцию**, например изменения поведения API, модификации контрактов, схем XSD, лимитов и т.п. Изменения, не влияющие на API и незаметные с точки зрения интеграции, например внутренние исправления качества, производительности, безопасности **могут не сообщаться** или будут представляться в сводном виде.
